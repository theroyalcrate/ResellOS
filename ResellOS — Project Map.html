// Known Risks and Mitigations
Keepa API
Single point of failure. API terms can change, Amazon can pressure Keepa. OS functions fully without it. Build 2 treated as separate product, not core dependency.
Tax treatment
CPA conversation required before Phase 3. How GWP, rewards, and cashback are treated for tax has real implications. Build the reporting to match CPA guidance, not assumptions.
Email parsing
Retailer email templates change without notice. Parser flags uncertainty rather than guessing. Wrong data in cost basis engine is worse than no data.
Scope creep
The map is larger than what's built. One session, one agent. Complete and verify before starting the next. Understanding over speed.
Historical data
15 months of orders exist outside the system. Migration planned for Phase 5. Don't try to migrate before the system is proven stable.
PostgreSQL migration
Most complex step in Phase 6. Budget extra time. SQLite and PostgreSQL are similar enough that agents barely change — but the migration itself needs careful testing.
Cert expiry
Reseller certificate has an expiry date. System alerts on approach. Lapsed certificate means tax recovery stops accruing. High priority reminder built into Phase 5.
