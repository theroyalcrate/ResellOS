🧮The Four-Layer Cost Basis Model

Every set in inventory has a true cost basis that reflects all four layers. Not what the invoice says. What it actually cost to have that set available to sell. This number changes every sell decision.

Layer What It Is How It Works Example — LEGO Order
1. Invoice Total Base purchase cost including tax Auto-populated from invoice parsing $171.03
2. Gift Card Discount Savings from buying gift cards below face value % discount entered per order — typically 10% but variable -$17.10 (10%)
3. Retailer Rewards Points/cash earned in retailer loyalty program Rate set per order (default 5%, adjustable for events). Flows to rewards pool. Allocated when spent. -$7.75 (5%)
4. External Cashback Rakuten or similar cashback on pre-tax spend Rate set per order. Flows to cashback pool with payout tracking. Allocated when received. -$3.87 (2.5%)
5. GWP Net Proceeds Actual net from selling free gift-with-purchase sets Entered when GWP sets sell. Allocated proportionally across paid sets on that order. -$85.38
True Net Cost What the paid sets actually cost after all layers $56.93
🏦The Rewards Pool System
  • Earn side: Every order calculates rewards at the order-specific rate and adds to the retailer's pool. Rate defaults to 5% but is adjustable per order for promotional events (10%, 20%, custom).
  • Spend side: When rewards are used on a future order, they're drawn from the pool and allocated proportionally across that order's sets as a cost reduction.
  • Thread preserved: The connection between the earning order and the spending order is maintained. Full history visible at any time.
  • Pool balance: Running balance of earned but unspent rewards across all retailers — treated as a real asset in the business.
  • Separate pools per retailer: LEGO Insider, Kohl's Cash, B&N Rewards, Macy's Star Rewards — each has its own pool since rewards are earned and spent within each ecosystem.
💰The Cashback Pool System
  • Earn side: Rakuten cashback rate set per order (2.5% base, up to 15% during promotions). Calculated on pre-tax amount. Added to cashback pool.
  • Payout tracking: Rakuten pays quarterly. Pool tracks earned vs. received with expected payout quarter.
  • Allocation on receipt: When cashback hits your account, it's allocated back to the orders that earned it and adjusts cost basis on those sets.
  • Stacks with retailer rewards: Rakuten applies on top of native retailer rewards — both are captured independently.
  • Expandable: Architecture supports adding other cashback platforms beyond Rakuten.
